Deploy and Install XDA

WHAT DOES IT DO?

To install XDA:

  1. Run wget bda-xda-<version>.rpm from where the rpm file is located and press Enter.

    2. Notes:

    • Enter the current file name in place of <version>.
    • For example: wget https://synchross/downloads/bda-xda-3.0.0.rpm.
    • Refer to Deployment Versions and Packages.
  1. Run the sudo su - command and press Enter.
  2. Enter the Password and press Enter.
  3. Run cd /etc/bda and press Enter.
  4. Run ll and press Enter to verify that xda-act-spr.vars is present. If the vars file is not present, continue and if the vars file is present, proceed to step 8.
  5. Run sudo /dfs/opt/bda/apps/bda-xda-<version>/bin/appl_activate.sh.
  6. Run sudo cp xda-act-spr.vars.changeme xda-act-spr.vars and press Enter.
  7. Run sudo vi xda-act-spr.vars and press Enter.
  8. Type i to edit the file.
  9. Configure the xda-act-spr.vars file. For example:

Note: Replace CHANGEME # with appropriate values similar to the values given after #. Replace the entire line including # values with appropriate values.

Example:

  • Right way: spark.master = spark://<maprnode103 FQDN>:7000
  • Wrong way: spark.master = # spark://<maprnode103 FQDN>:7000
# This is vars file for XDF application
# To be copied to /etc/bda/xda-act-spr.vars
# after altering the changeme values

spark.master = changeme            # Add Spark master value of server
driver.memory = changeme            # Add Driver memory - 2g
spark.cores.max = changeme          # Add Spark Cores value checking the configuration of server - 6
spark.executor.memory = changeme    # Add Spark Executor memory value checking the configuration of server
es.batch.size.bytes = changeme      # ES Batch Size Bytes - Preferably 10mb
es.batch.size.entries = changeme    # ES Batch Entries - Preferably 10000
es.admin.port = changeme            # ES Admin Port - Preferably 9300
es.cluster.name = changeme          # ES Cluster Name
es.nodes = changeme                 # ES Node Name
es.net.http.auth.user = changeme   # ES X-Pack User name
es.net.http.auth.pass = changeme    # ES X-Pack Password
es.internal.es.version = changeme   # ES Cluster Version Number. Preferable 5.3.0
oozie_home = changeme               # Oozie Home Path
oozie_url = changeme                # Oozie URL to check Oozie job status
job_tracker = changeme              # Job Tracker URL setting. Values will be maprfs:/// if cluster is not on YARN. If YARN, give resource manager URL
name_node = changeme                # Name node value. Will be maprfs:///
maprfs_path = changeme             # maprfs home location on the server where application is to be installed. For example /mapr/bda_lab_batch

  1. Press ESC and type :wq! to save and exit.
  2. Run cd and press Enter.
  3. Run sudo rpm –i bda-xda-act-spr-<version>.rpm and press Enter. Again replacing the <version> as above.
  4. Run cd /dfs/opt/bda/apps and press Enter.
  5. Run ll and press Enter to verify that the xda-act-spr-<version> directory is present.
  6. Verify that the xda-act-spr soft link works. HOW?
  7. Run cd /dfs/opt/bda/apps/xda-act-spr-<version>/bin and press Enter. Again replacing the <version> as above.
  8. Run -l and press Enter.
  9. Check files permission for executables in bin folder as provided by your Deployment Specialist.
  10. Refer to Configure and Schedule Oozie Coordinator Job.
  11. Refer to Validate.
