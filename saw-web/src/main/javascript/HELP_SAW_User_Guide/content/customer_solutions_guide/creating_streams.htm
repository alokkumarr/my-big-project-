Create Streams

To create streams:

  1. Run su - mapr and press Enter.
  2. Run hadoop fs -mkdir -p "maprfs:///bda/data/streams" and press Enter to create MapR volumes and directories for streams.
  3. Create streams and set permissionsas the mapr user. For example:

    4. maprcli stream create -path /bda/data/streams/rta-xxx1

    maprcli stream create -path /bda/data/streams/rta-act-xxx

    maprcli stream create -path /bda/data/streams/rta-xxx2

    maprcli stream topic create -path /bda/data/streams/rta-xxx1 -topic data1

    maprcli stream topic create -path /bda/data/streams/rta-act-xxx -topic data1

    maprcli stream topic create -path /bda/data/streams/rta-xxx1 -topic data2

    maprcli stream topic create -path /bda/data/streams/rta-xxx2 -topic data1

    maprcli stream topic create -path /bda/data/streams/rta-xxx2 -topic data2

    maprcli stream edit -path /bda/data/streams/rta-xxx1 -produceperm u:rtis-user -consumeperm u:mapr -topicperm u:rtis-user

    maprcli stream edit -path /bda/data/streams/rta-act-xxx -produceperm u:rtis-user -consumeperm u:mapr -topicperm u:rtis-user

    maprcli stream edit -path /bda/data/streams/ rta-xxx2 -produceperm u:rtis-user -consumeperm u:mapr -topicperm u:rtis-user

    Note: Ignore any Warnings.
  4. Run sudo mkdir /etc/bda and press Enter.
  5. Run cd /etc/bda and press Enter.
  6. Copy the cluster.vars code:
    7. # This file is stored here for reference only,
    # edited version of this file must be present in /etc/bda
    # for package installation.
    ##
    # Values for Charter-sc application
    app.key = CHANGE_ME # ACT30.xxxx
    streams_1.topic = CHANGE_ME # data
    streams_1.queue = CHANGE_ME # bda/data/streams/rta-act-spr
    streams_2.topic = CHANGE_ME # data
    streams_2.queue = CHANGE_ME # bda/data/streams/rta-act-spr
    app.class = CHANGE_ME # synchronoss.handlers.GenericEventHandler
    key.serializer = CHANGE_ME # org.apache.kafka.common.serialization.ByteArraySerializer
    value.serializer = CHANGE_ME # org.apache.kafka.common.serialization.ByteArraySerializer
  7. Run sudo vi cluster.vars and press Enter.
  8. Type i and press Enter.
  9. Paste the code.

    10. Note: Replace CHANGEME # with appropriate values similar to the values given after #. Please replace the entire line including # values with appropriate values.

    Example:

    • Right way:app.key = ACT30.xxxx
    • Wrong way:app.key = CHANGE_ME # ACT30.xxxx
  10. Replace CHANGE ME # with the correct information. For example:
    11. app.key = ACT30.xxxx
    streams_1.topic = data
    streams_1.queue = bda/data/streams/rta-act-spr
    streams_2.topic = data
    streams_2.queue = bda/data/streams/rta-act-spr
    app.class = synchronoss.handlers.GenericEventHandler
    key.serializer = org.apache.kafka.common.serialization.ByteArraySerializer
    value.serializer = org.apache.kafka.common.serialization.ByteArraySerializer
  11. Press ESC and type :wq! to save and exit.

For problem investigations, check log files in /opt/bda/apps/project/var/log directory. Refer toTroubleshooting.
