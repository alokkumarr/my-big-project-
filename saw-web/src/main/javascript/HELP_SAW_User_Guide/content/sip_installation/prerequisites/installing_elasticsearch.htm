Elasticsearch

ElasticsearchElasticsearch is a distributed, RESTful search and analytics engine capable of solving a growing number of use cases. Elasticsearch centrally stores data for discovery. is a distributed, RESTful search and analytics engine that centrally stores data for discovery. Refer to Elasticsearch.

Note: Install Elasticsearch on EACH data collection node.

To install Elasticsearch:

  1. Refer to Clusters and Nodes and Create Users and Groups.
  2. Install Java 1.8.
Note: Acquire the tar file from the FTPFile Transfer Protocol./Java official website/other environments to a /tmp location. Refer to http://www.oracle.com/technetwork/pt/java/javase/downloads/jdk8-downloads-2133151.html.
  1. Run mv /tmp/jdk /usr/lib/jvk and press Enter.
  2. Run cd /usr/lib/jvm/ and press Enter.
  3. Run tar –zxvf jdk-8u151-linux-x64.tar.gz and press Enter.
  4. Run cd jdk1.8.0_151/ and press Enter.
  5. Run alternatives --install /usr/bin/java java /usr/lib/jvm/jdk1.8.0_151/bin/java 1 and press Enter.
  6. Run export JAVA_HOME=/usr/lib/jvm/jdk1.8.0_151/ and press Enter.
  7. Run alternatives --config java and press Enter.
  8. Select the correct java version and press Enter.
  9. Run java –version and press Enter.
  10. Run cd /etc/yum.repos.d/ and press Enter.
  11. Run vi elasticsearch.repo and press Enter to create an elasticsearch.repo.
  12. Add the following:
[elasticsearch-5.x]
name=Elasticsearch repository for 5.x packages
baseurl=https://artifacts.elastic.co/packages/5.x/yum
gpgcheck=1
gpgkey=https://artifacts.elastic.co/GPG-KEY-elasticsearch
enabled=1
autorefresh=1
type=rpm-md
  1. Press ESC and type :wq! to save and exit.
  2. Run yum install elasticsearch-5.3.0-1.noarch and press Enter.
  3. Run cd / and press Enter.
  4. Run mkdir elasticsearch and press Enter.
  5. Run chown elasticsearch:elasticsearch elasticsearch and press Enter.
  6. Run cd /etc/elasticsearch and press Enter.
  7. Run vi elasticsearch.yml and press Enter.
  8. Edit following values:
cluster.name: CLUSTERNAME
node.name: HOSTNAME      #elk01.ana.qa.vaste.sncrcorp.net
path.data: /elasticsearch
network.host: CHANGEME      #elk01.ana.qa.vaste.sncrcorp.net
http.port: 9200
discovery.zen.ping.unicast.hosts: ["172.31.208.XX", "172.31.208.YY", "1172.31.208.ZZ"]
  1. Press ESC and type :wq! to save and exit.
  2. Run service elasticsearch start and press Enter.
Note: You can verify the page at http://hostnameorip:9200 if the installation is successful.
  1. Repeat for all the Elasticsearch nodes.

Troubleshooting Elasticsearch

If you encounter a “bootstarp checks failed” errors in logs while starting Elasticsearch at /var/log/elasticsearch/.

  1. Run cd /etc/security and press Enter.
  2. Run vi /etc/security/limits.conf and press Enter.
  1. Edit following values:
elasticsearch - nofile 65536
elasticsearch - nproc 4096
  1. Press ESC and type :wq! to save and exit.

If you encounter further Elasticsearch startup errors:

  1. Run cd /etc/elasticsearch/ and press Enter.
  2. Run vi /etc/elasticsearch/elasticsearch.yml and press Enter.
  3. Update the bootstrap lines:
#
# Lock the memory on startup:
#
bootstrap.memory_lock: false
bootstrap.system_call_filter: false
  1. Press ESC and type :wq! to save and exit.

Elasticsearch File Locations

The following are critical locations:

  • ## Configuration file:

/etc/elasticsearch/elasticsearch.yml

  • ## Binary/Libraries:

/usr/share/elasticsearch

  • ## Log files:

/var/log/elasticsearch

Note: Elasticsearch does not support data placed on MapR-FS.
