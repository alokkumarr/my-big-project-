Deploy RTIS

Events are ingested real-time using Real Time Ingestion Service (RTIS) frameworks and then the XDF pipeline framework processes the data.

Note: Install and deploy RTIS on EACH node.
  1. Refer to Clusters and Nodes and Create Users and Groups.
  2. Run su - mapr and press Enter.
  3. Run wget bda-rt-<version>.rpm from where the rpm file is located and press Enter.

    4. Notes:

    • Enter the current file name in place of <version>.
    • For example: wget https://synchross/downloads/RT/bda-rt-1.2.1.rpm.
    • Refer to Deployment Versions and Packages.
  1. Run vi /etc/bda/yum.cluster.vars and press Enter.
  2. Run vi /etc/bda/cluster.vars and press Enter.
  3. Type i to edit the file.
  4. Add the following:
app.key = <projectname>.PROJECT
streams_1.topic = data
streams_1.queue = bda/data/streams/project
streams_2.topic = data
streams_2.queue = bda/data/streams/project
app.class = synchronoss.handlers.GenericEventHandler
key.serializer = org.apache.kafka.common.serialization.ByteArraySerializer
value.serializer = org.apache.kafka.common.serialization.ByteArraySerializer
  1. Press ESC and type :wq! and press Enter to save and exit.
  1. Run sudo rpm -ivh bda-rt-<version>.rpm changing the file name to reflect the file in the wget command and press Enter.
    2. Note: Run the installation RPM on EACH RTIS host (rtfeXXX).
  2. Copy the following code to /opt/bda/rtis/conf/application.conf:
    3. stream.topic.selectormethod=random
  3. Run vi /opt/bda/rtis/conf/application.conf and press Enter.
  4. Paste the stream.topic.selectormethod=random code at the end of the file.
  5. Press ESC and type :wq! to save and exit.
  6. Repeat for each RTIS host.
  7. Copy the following code to /etc/rc.local:
    8. # start BDA front end server
    /opt/bda/rtis/sbin/rtis_runner.sh &>/tmp/rc.local.rtis_runner.log
  8. Run sudo vi /etc/rc.local and press Enter.
  9. Paste the previous code at the end of the file.
  10. Press ESC and type :wq! to save and exit.
  11. Repeat for each RTIS node.
Note: Whoever actively monitors the Nagios dashboard and notifies the correct team via email, Slack, or other means if they see any alerts on the dashboard.

On each RTIS node execute the following:

  1. Run su - rtis-user and press Enter.
  2. Run /opt/bda/rtis/sbin/rtis_runner.sh and press Enter.
  3. Run ps -ef | grep rtis and press Enter to ensure the process is running.
  4. Run hostname -i and press Enter.
  5. Copy the IPIP Address Address and the Port Number.
  6. Open a browser.
  7. Paste the IP Address and the Port Number.

Your application should be ready.

For problem investigations, check log files in /opt/bda/apps/project/var/log directory. Refer to Troubleshooting.
