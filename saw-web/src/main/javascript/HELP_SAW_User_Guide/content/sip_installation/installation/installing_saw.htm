Deploy SAW

Each SAWSynchronoss Analytics Workbench environment requires its own configuration. A sample configuration has been provided in the config file, which defaults to installing on localhost. At a minimum, copy this sample file and change the localhost entries to the target host.

The sample configuration defaults to installing all available modules: SAW Web, SAW Services, and SAW Security. To install only a subset of these modules, edit the environment configuration to only include the modules you desire.

Caution: The environment configuration file must be preserved between installations and upgrades. It is recommended to put it under version control that is stored on the deploy host and backed up to another location.

The SAW Web module and supporting services are exposed on port 80 of the target host (http://<target-host>/). The SAW Web application automatically discovers the endpoints for SAW Security and SAW Services based on the URL from which it is being served. No other ports are accessed by external parties.

The SAW target host only accesses port 80 deployment.

Note: To upgrade an existing installation, follow the same install steps. The deploy command detects an already existing installation and upgrades it.

  12. Run rpm --import http://package.mapr.com/releases/pub/maprgpg.key to install the key on the SAW node.
  13. Run yum install mapr-client and press Enter.
  14. Type y and press Enter.

  16. Install and configure the MapR client on the SAW target host. Refer to Deploy and Install MapR.

  18. Run yum install mapr-spark and press Enter.

  21. Run rpm –ivh pm -ivh epel-release-<version>.rpm and press Enter.

  23. Run pip install ansible==<required version> and press Enter.

    2. Notes:

    • Target hosts must be allocated exclusively for SAW use. The underlying infrastructure up to the operating system level is to be managed externally, while SAW fully manages packages and configuration on the target hosts.

  1. Press ESC and type :wq! to save and exit.
  1. Run cd <saw location>/saw/ and press Enter.
  2. Run ./saw-deploy saw-config and press Enter to install SAW.
  3. Run service firewalld stop and press Enter.
  4. Open the browser and verify the status of the install at http://ip/web/, such as http://10.xx.86.192/web/.
Note: Configure passwordless SSH access to the target host for a smoother installation. Run the deploy command as a normal user. The deploy command uses sudo to request privileges for relevant operations.

Status

To check the status of all SAW systemd units, run sudo systemctl list-units --all saw-* and press Enter.

Note: It is normal for the saw-scheduler service to be in an inactive state. It is activated on a set schedule by saw-scheduler.timer.

To check the status of all SAW systemd timers, run sudo systemctl list-timers --all saw-* and press Enter.

Start, Stop, and Restart SAW Services

There should be no need to start, stop, or restart SAW services manually. If needed:

  • Run sudo systemctl start <saw-service> and press Enter.
  • Run sudo systemctl stop <saw-service> and press Enter.
  • Run sudo systemctl restart <saw-service> and press Enter.

Where <saw-service> is one of the SAW systemd services (for example saw-gateway). To list SAW services:

  • Run sudo systemctl list-units saw-\* and press Enter.

Logs

To view the SAW system logs:

  • Run sudo journalctl saw-\* and press Enter.

To view logs of individual services:

  • Run sudo journalctl -u saw-\* and press Enter.

Large Header Settings

Include these properties in the NGINX server config file to support, http request with large headers more than 8K.

  • client_body_buffer_size 32k;
  • client_header_buffer_size 16k;
  • large_client_header_buffers 8 64k;
