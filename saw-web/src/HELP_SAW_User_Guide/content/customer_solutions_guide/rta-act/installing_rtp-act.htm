Deploy and Install RTA-ACT

The Real-Time Application (RTA) collects data at real-time and writes raw data to the data lake for further processing. This application runs on top of Real-time Processing Service (RTPS) framework. The RTA-ACT (Activation) application continuously monitors the MapR stream. If data is available in the stream, RTA-ACT collects that data and writes the data to the data lake based on the given schema structure.

  1. Run su - mapr and press Enter.
  2. Run sudo su - and press Enter.
  3. Run cd /etc/bda/ and press Enter.
  4. Run ll and press Enter to verify the rta-act-spr.vars file is present. If not, proceed to Step 5. If it is present, continue to step 6.

    -rw-r--r--. 1 root root 974 Nov 7 14:18 rta-act-spr.vars.changeme

    lrwxrwxrwx. 1 root root 45 Nov 9 10:07 rtis.env -> /opt/bda/rtis-<version>/conf/rtis.env

    -rw-r--r--. 1 root root 1951 Nov 9 15:33 xda-act-spr.vars

    -rw-r--r--. 1 root root 1368 Nov 9 10:41 xda-act-spr.vars.changeme

  5. Run sudo cp rta-act-spr.vars.changeme rta-act-spr.vars and press Enter.
  6. Run sudo chown mapr:mapr rta-act-spr.vars and press Enter.
  7. Run cat rta-act-spr.vars and press Enter to display concatenate files. Note the information.
    8. spark.master =spark://<maprnode101 FQDN>:7077
    streams.directory =/dba/data/streams
  8. Run sudo vi rta-act-spr.vars and press Enter.

    9. Notes:

    • Change the values according to the environment.
    • The Spark.Master is the master node.
  1. Type i to edit the file.
  2. Enter the changes per the cat command above.

    3. For example, change the spark.master = CHANGE ME # to reflect the IPIP Address address, such as Spark.master = spark://10.xx.x.35:7077 or change the Streams.directory = CHANGEME #to Streams.directory = /bda/data/streams.

  3. Press ESC.
  4. Type :wq! and press Enter to save and exit.
  5. Run wget rta-act-spr-<version>.rpm from where the rpm file is located and press Enter.

    6. Notes:

    • Enter the current file name in place of <version>.
    • For example: wget https://synchross/downloads/rta-act-spr-3.0.0.rpm.
    • Refer to Deployment Versions and Packages.
  1. Run sudo rpm -ivh rta-act-spr-<version>.rpm and press Enter. Change the <version> as above.

    2. This installs the application under /dfs/opt/bda/apps/rta-act-spr-<version> and creates a symbolic link that points to the application home directory.

  2. Run chmod 0755 /opt/mapr/conf/conf.d and press Enter.
  3. Run cp /dfs/opt/bda/apps/rta-act-spr-<version>/conf/warden.rta-act-spr.conf /opt/mapr/conf/conf.d and press Enter.
  4. Run sudo /opt/mapr/server/confiugre.sh -R and press Enter.
  5. Run ll and press Enter.

    6. total 1

    lrwxrwxrwx. 1 root root 18 Nov 8 10:44 rta-act-spr -> rta-act-spr-2.0.0

    drwxr-xr-x. 4 mapr mapr  2 Nov 8 10:44 rta-act-spr-2.0.0
  6. Run cd /dfs/opt/bda/apps/rta-act-spr-<version> and press Enter.
  7. Run ll and verify the /bin and /conf directories are present on EACH MapR node. If not, reinstall the application.
  1. Run cd /dfs/var/bda/rta-act-spr-<version>/ and press Enter.
  2. Run ll and verify the /log and /run directories are present on EACH MapR node. If not, reinstall the application.
  3. Run cd /dfs/opt/bda/apps/rta-act-spr-<version>/conf and press Enter.
  4. Run sudo vi rta-act-spr.conf and press Enter.
  5. Verify the properties are correct on rta-act-spr.conf. WHAT ARE THEY?
  6. Run cp /dfs/opt/bda/apps/rta-act-spr-<version>/conf/warden.rta-act-spr.conf /opt/mapr/conf/conf.d and press Enter.
  7. Run cd /opt/mapr/conf/conf.d and press Enter.
  1. Run sudo chown mapr:mapr warden.rta-act-spr.conf and press Enter. I thought we copied it to the name conf.d?
  2. Run sudo vi warden.rta-act-spr.conf and press Enter. NAME
  1. Type i to edit the file.
  1. Configure the warden.rta-act-spr.conf file as instructed by your Deployment Specialist. For example:
    2. services=rca-acc-spr:1:cldb
    service.displayname=RTPS-rca-acc-spr
    service.heapsize.min=1000
    service.heapsize.max=3000
    service.logs.locacion=/dfs/var/bda/rca-acc-spr/log
    service.command.scarc=/dfs/opc/bda/apps/rca-acc-spr-2.0.O/bin/rca-acc-spr.sh scare
    service.command.scop=/dfs/opc/bda/apps/rca-acc-spr-2.0.O/bin/rca-acc-spr.sh scopservice.command.monicorcommand=/dfs/opc/bda/apps/rca-acc-spr-2.0.O/bin/rca-acc-spr.sh scacus service.command.cype=BACKGROUND
    service.alarm.cersename=rca-acc-spr
    service.alarm.label=rca-acc-spr.DOWNALARM

    # Doc: heep://doc.mapr.com/display/MapR/warden.%3Cservicename%3E.conf

  2. Press ESC.

  3. Type :wq! and press Enter to save and exit.

  4. Run cd conf.d and press Enter.
  5. Run ll and press Enter.

    6. The following configuration files should display:

    total 16
    -rw-r- -r—. 1 mapr mapr 481 Oct 31 11:35 warden.drill-bits.conf
    -rw-r- -r—. 1 mapr mapr 1031 Aug 1 20:27 warden.nodemanager.conf
    -rw-r- -r—. 1 mapr mapr 1452 Oct 31 11:25 warden.resourcemanager.conf
    -rw-r- -r—. 1 mapr mapr 656 Aug 27 02:23 warden.spark-master.conf

  6. Run sudo /opt/mapr/server/configure.sh -R and press Enter.

  7. Open a browser and enter your IP Address. For example, https://10.49.3.xx:8443.
  8. Log in to MapR and the dashboard should display. Refer to Prepare MapR.
  9. Run cat rta-act-spr.vars and press Enter to display concatenate files. Note the information.
    10. spark.master =spark://<maprnode101 FQDN>:7077
    streams.directory =/dba/data/streams
  10. Enter the Spark Master URL in the browser to verify the application is running. Refer to Prepare MapR.
Note: If the application starts successfully, “OK started” displays on the console. If the application does not automatically start, refer to Manually Start and Stop RTA-ACT.

Refer to /opt/mapr/logs/configure.log for issue resolution. Refer to Troubleshooting.
