4.4.2. Add a Privilege

 

To add a Privilege the following information must be provided.  All fields marked with an (*) are required:

  • Product*
  • Role*
  • Module*
  • Category

 

How to Add a New Privilege

  1. From the 'Admin' Module (refer to section 4.1. Access the Admin Module), select the menu icon located on the upper left-hand side of the screen

Figure 32 Admin Selection

  1. Select to populate the Add New Privilege screen
  2. Select the 'Product*' from the drop-down that needs the Privilege
  3. Select the 'Role*' from the drop-down that will have access to the product (Admin or one of the User Roles)
  4. Select the 'Module*' related to the Product (Analyze, Observe, Workbench, etc…)
  5. Select the relevant 'Category' (refer to section 4.5. Category Management)

Figure 33 Add New Privilege Screen

Figure 34 Add a Privilege with Sub-Categories Screen

NOTE: Once all fields are entered, the sub-categories are available for selection

 

  1. Select the Sub-Categories for Access and Privileges:
    1. Access - Turn on to enable the Privilege to perform the following
      1. All (select to Create, Edit, and Fork all Privileges)
      1. Create – Create Privilege
      2. Execute - Run and refresh the results in Analyze module to acquire the latest data for an analysis
      3. Publish - Publish the data to a shared sub-category. Publish enables you to move an item to a different sub-category. Publish also gives you the ability to create, edit, remove a schedule for automatic recurring execution.
      4. Fork – Copy Privileges
      5. Edit – Edit existing Privileges
      6. Export - Export data and download it your computer via the browser. The format depends on where you are in the application: CSV for reports and Excel (XLSX) for pivot data.
      7. Delete – Delete the data
  2. Select Create Privilege.
