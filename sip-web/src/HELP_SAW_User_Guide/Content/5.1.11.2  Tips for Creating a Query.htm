5.1.11.2 Tips for Creating a Query

 

  • Selecting all possible columns in designer mode before switching to ‘Query’ mode can be a significant time saver and reduce the possibility of errors. This is especially true when creating filters such as MTD or YTD. Shown below, adding MTD filter for Event Date and the SQL that is generated

Figure 96 Adding MTD Filter for Event Date

Figure 97 SQL Generated for Adding MTD Filter for Event Date

  • Save a designer mode version of the report in “My Analysis -> Draft folder and then re-open by selecting “Edit”, then change to Query Mode. This will enable the user to quickly revert to the saved version if they are experiencing issues working with SQL.
  • Filter the query to return as little data as possible until the query is working as expected. A useful filter to add to SQL is limit n where n is the number of rows to return.
