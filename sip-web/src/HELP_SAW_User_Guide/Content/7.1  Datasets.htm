7.1 Datasets

A Data Set points to a set of files in a data source and describes the structure of the data, as well as any processing logic needed to prepare the data for consumption. It is a metadata description of the data that contains all of the data related to the data -- plus a sampling of raw rows to facilitate data discovery.

A Data Source is a connection to a mount point or directory on an external data server, such as a file system or database server.

To create a data Set, the user needs access to the source data via the data source. Before a new data Set is added to the workbench, the source data files on which the data Set is based must be in the supported repository and accessible via a Data Source. A user with appropriate permissions can also upload files from their desktop or accessible location to the default Uploads data source.

The Workbench Datasets screen contains several options available that the user with appropriate permissions can access:

  • Data Sets active selection
  • List mode or a card mode option
  • Data Pods (not active when gray) selection
  • The user can decide if they would like the data to be auto refreshed by selecting the Auto Refresh toggle switch or not

  • Search by clicking on the magnifying glass icon and enter the data set name in the field that is displayed
