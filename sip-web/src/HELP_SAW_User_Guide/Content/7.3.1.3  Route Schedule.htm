7.3.1.3 Route Schedule

 

A route is typically associated with a schedule to allow periodic pull of data from the source channel. A route should be scheduled with one of the following options

  • Immediate
  • Hourly (Including Minutes)
  • Daily
  • Weekly
  • Monthly
  • Yearly

All transferred files are stored in a destination folder that is named with a specific naming convention (Batch Folder Format: YYYYMMDDHHMMSS)

 

How to Schedule a Route

 

  1. Continue from step 4 of 7.3.1.2. Routes
  2. The next step is to apply the schedule for the route

  1. Select from one of the following options:
    1. Immediate – this is to immediately run data without any recurrences (refer to 7.3.1.3 Route Schedule)
    2. Hourly – Schedule for every “number of hours” hours on the minute to run the data (refer to 7.3.1.3 Route Schedule)
    3. Daily – A choice can be made to schedule the day of the week (1 – 31) at a specific time or select to schedule every day of the week. In addition, a date can be provided of when the run should start (Start Date) and end (End Date) (refer to 7.3.1.3 Route Schedule)
    4. Weekly – A day or multiple days of the week can be selected to run the data, along with the time to start the run and the dates the process should start and end (refer to 7.3.1.3 Route Schedule)
    5. Monthly – Select to run monthly and when to run during the month (refer to 7.3.1.3 Route Schedule)
    6. Yearly - Select to run monthly and when to run during the month (refer to 7.3.1.3 Route Schedule)
