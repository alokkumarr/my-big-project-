7.2 Datapods

 

Users with appropriate permissions will have the capability to access Data Pods. Data Pods can be added viewed in list or card mode as displayed in the following instructions and screen shots.

A Data Pod is a type of data storage that is specific to the workbench module. This application uses Hadoop data lake or ElasticSearch as its data source to build and store its data pods. Once a data pod is built, this prepared data is available in the Analyze module of SAW, where it is then available for analysis. A data pod is considered as a dynamic on-demand data mart, built for a specific analysis project.

The Workbench Datapods screen contains several options available that the user with appropriate permissions can access:

  • Data Pods active selection
  • List mode or a card mode option
  • Data Sets (not active when gray)
  • The user can decide if they would like the data to be auto refreshed by selecting the Auto Refresh toggle switch or not

  • Datasets can be added by clicking

  • Search by clicking on the magnifying glass icon and enter the data set name in the field that is displayed

 

How to View Data Pods

 

  1. Log into Saw
  2. Click on the Module Icon to display the modules available for selection
  3. Click on Workbench, to access the ‘Workbench’ module

  4. Click on the menu icon from the upper left-hand side of the screen

  5. Click on ‘Datasets’ drop-down menu from the menu options

  6. Click on ‘View Datasets’, the default screen will be displayed

  7. Click on Data Pods button on the screen
