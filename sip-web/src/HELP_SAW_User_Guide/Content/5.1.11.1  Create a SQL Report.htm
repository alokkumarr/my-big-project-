5.1.11.1 Create a SQL Report

 

The following are the steps to assist in creating a SQL Report.

How to Create a New Analysis

  1. Select the Module Icon from the upper right hand corner of the screen

  2. Select the Analyze module
  3. Select the Add Analysis button from the upper right-hand side of the screen
  4. Select Report as Analysis Type

Figure 87 Analysis Type-Report

  1. Select the Next button

  2. Select a ‘Datapod’ based on Data Lake, which is typically identified by the (DL) suffix

  3. Select the Create button

Figure 88 Datapod

  1. The report editor window opens in Designer mode by default. This enables the creation of tabular reports by selecting available columns from displayed data objects
  2. Select the columns from the displayed data objects as displayed in Figure 90 Header Band Column Drop-Down Menu

Figure 89 Designer Mode

  1. Columns are automatically added to a header band as they are selected.
  2. From the header band, columns can be reordered by dragging to the left or right of other columns. Columns may also be renamed, or selected as summary or grouping fields by clicking the pull down menu to the right of the column name as shown in Figure 90 Header Band Column Drop-Down Menu

Figure 90 Header Band Column Drop-Down Menu

  1. To view the data that will be returned click ‘Refresh data’ or to view what the report will look like click ‘Preview’ as in Figure 91 Sample Report

Figure 91 Sample Report

When creating report in Designer mode, SIP automatically generates a SQL statement that will be executed against the underlying data table(s). To see the generated query

  1. Click the ‘Query’ tab at the top of the editor

Figure 92 Query Tab Selected

From here, it is possible to modify the query to produce the desired result set.

Display Names vs Column Names

When editing SQL, the column name selections will display the business friendly names from the metadata model. However, once a column is selected, the physical COLUMN_NAME from the underlying table is added to the SQL as displayed in Figure 94 Business Friendly Names from Metadata Model.

Figure 94 Business Friendly Names from Metadata Model

  1. Click to view the results of the SQL

If you have converted from designer to query mode, the following warning will appear:

Figure 95 Designer to Query Mode Warning Message

  1. Click Save to continue. The Query will execute and the results will be displayed.
